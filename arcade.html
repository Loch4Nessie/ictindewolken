Project 1 - Arcade kast

Het project

Voor de eerste periode hebben Mees en ik besoten om een arcade kast te bouwen en een game te maken
die je op die kast kan spelen. Voor de arcade kast was ons idee om een de kast meer een controler te maken waarmee je
op elke monitor je games kon spelen. Deze controler hadden we ontworpen in blender en het ontwerp werd ge-3D-print.
Voor de game hadden we het idee om een 2D platformer game te maken in p5js. Tijdens dit project ging Mees vooral aan de
slag met de hardware en ik hield me vooral bezig met de game.

Hoe het is gegaan

Voordat Mees en ik wilden dus meer een portable controler maken als arcade kast. Dit betekent ook dat we een andere
joystick en andere knoppen nodig hadden. Het duurde dus even voordat we aan de slag konden gaan met de hardware dus
gingen we eerst plannen maken voor de game. Het originele idee was om een platformer te maken die allemaal stages moet
halen om uiteindelijk bij de eindbaas te komen. Uiteindelijk hadden we geen tijd meer om een eindbaas te progammeren
maar de rest van het originele idee is redelijk constant gebleven. We begonnen met de code van een eerdere game die
ik samen met Dries in de 4e had gemaakt, Dümpell. Ik vond het wel leuk om te zien dat ik alle code van twee jaar geleden
op een veel efficientere, makkelijkere en overzichtelijkere manier heb geschreven. Wat ik wel in de eerste game moest doen
en wat ik heel graag niet wilde hebben voor deze game was dat ik uren lang honderden regels moest copiëren en plakken waardoor
de code nauwlijks te volgen wordt. Dit zorgde er wel voor dat ik een stuk langer bezig was met het opzetten van een physics engine.
Het idee was hier om het scherm zeg maar in blokjes te verdelen en dat je in de code heel makkelijk en efficient levels kan maken.
Hier was ik heel lang erg veel problemen tegen gekomen omdat er de hele tijd net één ding niet klopte. Het bleek uiteindelijk
te zijn dat ik ergens in plaats van een 25 een 50 had opgeschreven. Al deze problemen werden uiteindelijk pas één week voor
de presentaties opgelost, dus van dat moment konden we dus pas echt de game zelf maken (zoals alle levels en alle blokjes).
We hebben ook nog een spooje weten toe te voegen die als je naar hem kijkt stil blijft staan en als je de andere kant op kijkt
hij achter je aan komt. Het probleem met het spookje was dat als je dood gaat door het spooje of als je de stage met het spookje
verlaat zal het spookje uiteindelijk weer spawnen op de plek waar hij als laatst was in plaats van waar hij eigenlijk zou moeten
spawnen. We hebben veel gebruik gemaakt van pixelart die we zelf uploaden als afbeelding in de game zodat er niet veel code aan
het maken van pixelart gaat. Als je de code van de game wil bekijken kan dat via deze link.

Voor de arcade kast ging het een stuk minder lekker tijdens deze periode. We hadden pas na drie weken alle onderdelen die we nodig
hadden om een goed ontwerp te kunnen maken voor de controler. Het maken van de controler in blender heeft ook wel een tijdje geduurt
en wachten tot het was uitgeprint duurde nog langer. Uiteinderlijk bleek ook dat de maten niet helemaal klopte zoals wij wilden
maar de controller was goed genoeg om verder te kunnen. het bleek wel dat de bovenkant van de cntroler te dun was en dat hij te
klein was om goed op de controler te passen waardoor we die niet kunnen gebruiken. Het grootste probleem was uiteindelijk dat de
joystick die wij hadden ander werkte dan de joystick die de anderen hadden. De normale joysticks gaven alleen waardes voor bijvoorbeeld
rechts, terwijl degene die wij hadden ook waardes had voor een beetje rechts en heel erg rechts. Dit heeft voor problemen gezorgt.
Helaas was op dit punt onze tijd op en konden we de controler niet optijd af krijgen. Wel willen we nog proberen om de controler
af te maken en de game op die controler te zetten.
